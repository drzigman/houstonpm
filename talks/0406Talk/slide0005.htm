Regex Modifiers /gsimexoc

The behaviour or interpretation of regular expressions can be modified through the use of several single-character modifiers that can be placed after the final delimiter of a match or substitution operator. Here is a brief description of these modifiers: 

    
    /i  pattern matches are case insensitive

    /m  multi-line mode: ^ and $ can match at the start or end of any
        line within the string (as opposed to the default behavior of
        matching only the start or end of the string).

    /s  allows the dot, ".", to match anything including a newline (the
        default is to not match newlines).

    /x  extended mode: allows a pattern to be written across multiple
        lines -- whitespace and comments are ignored.

    /o  the pattern is compiled only once -- this only makes sense if
        you use a variable in your pattern and that variable won't
        change.

    /g  global match: this will find all occurrences of the pattern
        in the target string.

    /c  do not reset search position when a /g match fails.

    /e  treat the replacement side of a substitution operation as
        Perl code to be evaluated (as opposed to treating it as a
        double quoted string).
