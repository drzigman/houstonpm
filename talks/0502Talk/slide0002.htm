2 Why Test?

    Any non-trivial program has bugs.
    Testing is one way to flush out the bugs.
    If the code isn't tested, you don't know it works.
      Beware of bugs in the above code; I have only proved it correct, not tried it. -- Donald Knuth
    Testing to prove code right.
    Testing to prove code wrong.
    Testing to improve confidence in code.
