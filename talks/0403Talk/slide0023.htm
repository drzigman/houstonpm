-p option

Looking at the -p option, we get a little more complicated. The specification portion ("-p <pattern>") tells us this option expects an argument, and that inside any action portion we can refer to this argument as the variable $pattern. In the description portion we also see the "[required]" directive. There are a few directives that can be used (all listed in the module documentation), and this one means that this option must be present on the command line. The action for this option sets a global variable to hold the pattern.
