14 Error Handling

The DBI adds a very useful feature for controlling the amount of error handling that you need to do.

    PrintError - display a message using warn on error and continue. (default)
    RaiseError - die with an message on error.
    Explicit error handling (not recommended).

These modes are enabled through the use of an optional reference to hash argument at the end of a DBI call. If used when creating a handle, the error checking mode is set for all methods called through that handle.

So by using, 

    
    my $dbh = DBI->connect( "dbi:CSV:f_dir=db", "fred", "secret",
                            { RaiseError => 1 }
			  );

We can make all errors into fatal errors and remove the need for explicit error checking. (Good for a quickie script.) Need better handling for a real program.
