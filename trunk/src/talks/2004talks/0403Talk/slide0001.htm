Parsing Command Line Options - manual option processing

Command line options provide a way for your program to offer flexible behavior under various usage conditions. For example, the perl program offers several different options that control how it behaves: 

    
    perl -c program   <- compile program only, do not run program
    perl -w program   <- compile and run program with warnings
    perl -e 'code'    <- treat 'code' as the script

See the perlrun pod-page for the complete list of command line options and their effects.

Back in June of 2003, I gave a talk on the 'faqgrep' program. That program allows for two command line options to control its behavior (-f and -or). The option processing loop in that program is the type of manual processing we'll look at in this section (We'll discuss other means of option processing in following slides).
