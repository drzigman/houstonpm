12.5 Output Explanation

The first line tells how many tests we expect to run. 

    
  1..6

Each successful test prints ok and the test number (plus an optional test name). 

    
  ok 1 - empty list

Each unsuccessful test prints not ok and the test number (plus an optional test name). 

    
  not ok 2 - max item is last

Any line starting with # is a comment and may be used for further explanation. 

    
  #     Failed test (t/max.t at line 8)
  #          got: '5'
  #     expected: '2'
