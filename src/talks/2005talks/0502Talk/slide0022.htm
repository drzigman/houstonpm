15 But What About Scripts?

    Test::Harness makes module testing easier.
    You can separate your script into a module and a driver script.
      This allows you to reuse the module code in other scripts.
      May be more work than is warranted.
    You can turn your script into a module.
      Move the script code into a subroutine like run or main.
      Call that routine with the following trick. 
    
   __PACKAGE__->run( @ARGV ) unless caller();
