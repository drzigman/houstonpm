Catalyst

  • Maybe the most popular perl web framework
  • There is tree based documentation
    • The Definitive Guide to Catalyst from Apress
  • MVC Based (Model View Control)
  • Catalyst provides good defaults but there are hundreds of plugins on the CPAN
    • Default Model DBIx::Class
    • Default View Template Toolkit
  • Helper tools
    • catalyst.pl *_cgi.pl *_create.pl *_fastcgi.pl *_server.pl *_test.pl
  • It's also very easy to extend
[any material that should appear in print but not on the slide]

Jifty

  • Very opinionated software
  • Defaults has one way to do everything
  • From the POD Documentation:
    • Out of the proverbial box, Jifty comes with one way to do everything you should need to do: One database mapper, one templating system, one web services layer, one AJAX toolkit, one set of handlers for standalone or FastCGI servers. We work hard to make all the bits play well together, so you don't have to.
  • That said they support two views for templating:
    • Template::Declare which is prefered
    • HTML::Mason
  • Helper tools
    • jifty ./bin/jifty
  • Many Plugins are on the CPAN

Mojolicous

  • Back Story
    • Written by Sebastian Riedel
    • He took over Maypole when Simon Cozens became a missionary
    • he wanted to make incompatible changes to Maypole
    • Started writing Catalyst
    • At version 4 of Catalyst he left to do rails work
    • He has now come back to perl with a new framework based on Sinatra
  • Mojolicous::Lite is a micro-framework
  • Has a REST base interface for a controller
  • Has a built in template language
  • Code runs from a single file
  • Template
    • Mojo::Template
    • Template::Toolkit
    • HTML::Mason
  • Helper tools
    • mojolicious

Dancer

  • Also base on the ruby project Sinatra
  • Also another micro-framework, but Mojolicious::Lite seems a little smaller
  • Templates
    • Dancer::Template::Simple
    • Template::Toolkit, by default uses '<% %>' as start and stop tags
  • Helper tools
    • dancer
