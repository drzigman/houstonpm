Description Of Example

Briefly, we first define our specification for our option parsing in the form of a single quoted here-doc (a multi-line single quoted string is fine as well). Each option specification is comprised of three main segments: the option pattern itself, a description, and optionally an action. The format is very specific: 

    
    option_pattern <tab> description [optional directive]
                         {action}

This entire specification is passed to the Getopt::Declare->new() function which builds a parser according to the specification and returns an object we can use to access the various options (we do not directly access them in the present example).
